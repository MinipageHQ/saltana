# Développeurs

# Obtenez vos clés APIs

Stelace utilise votre clé API pour authentifier vos demandes. Vous trouverez une clé API secrète et une clé publiable pour les environnements Live et Test dans a section Développeurs de votre tableau de bord. Pour obtenir votre clé API privée complète, il vous suffit de cliquer sur l’icône permettant de révéler la clé secrète.

  • Les clés API secrètes permettent aux développeurs ou aux applications d’effectuer des actions sans être authentifiés en tant qu’utilisateur. Cette clé secrète offre un accès complet grâce au rôle Doc dev, accordant l’ensemble des permissions de l’API. Vous pouvez l’utiliser pour développer plus rapidement. Assurez-vous de ne l’utiliser que dans des environnements sécurisés. Il est vivement recommandé de créer des clés API avec des autorisations restreintes si vous n’avez pas besoin de toutes les autorisations.

  • Les clés API publiables ne servent qu’à identifier votre plateforme. Elles ne permettent l’accès qu’aux points de terminaison de l’API (endpoints) publics. Pour accéder à des points de terminaison protégés, un jeton d’authentification (token) ayant les autorisations appropriées est requis: les utilisateurs doivent s’authentifier avant d’accéder à des ressources protégées.

# Configurez votre plateforme

L’identifiant de votre plateforme vous est communiqué via la vue Configuration. Cet identifiant est requis lors du paramétrage de l’authentification SSO Doc.
